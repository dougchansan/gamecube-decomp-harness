Method

Data source: projects/melee/state/orchestrator.sqlite, using runs, epochs, epoch_targets, target_claims, worker_state, and worker_checkpoints. Worker self-claims are not counted independently.

The report counts the first confirmed exact/improved checkpoint once per worker_state. Later checkpoints after a first improvement are analyzed separately in the path tables. Running workers are censored: their observed late successes are shown, but terminal-only probabilities exclude them.

The immediate prior epoch is included literally. If it has no confirmed successes, the latest prior result-bearing epoch is also included so the prior matches/improvements are visible with the same table shape.
