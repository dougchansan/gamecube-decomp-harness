# Past PR Source

Wraps the existing PR dump and synthesizes file-to-PR graph edges, PR search
chunks, review-risk hints, and file rollups.

The actual PR resources are co-located here:

- `data/prs/pr-NNNN/raw` - fetched GitHub PR metadata, comments, reviews, and
  diffs for one PR.
- `data/prs/pr-NNNN/extracted` - deterministic text, changed-file, diff-line,
  human-text, and review-comment slices for that PR.
- `data/prs/pr-NNNN/postmortem` - the structured model-reviewed or scaffolded
  postmortem for that PR.
- `data/aggregate` - corpus-wide rollups rebuilt from the per-PR slices.
- `data/library` - search-facing indexes, known fixes, and run summaries built
  from the PR-local postmortems.

This source slice is the bridge between PR evidence and the file graph.

Operational notes:

- `bun run pr:refresh:all` fetches the full PR corpus with 32 fetch workers and
  32 PR-review workers.
- `data/prs.json` is the discovered PR set for the active corpus.
- `data/aggregate/*.jsonl` is regenerated after fetch and before graph rebuild.
- `data/library/index.jsonl` is the postmortem library consumed by graph ingestion.
- `knowledge/sources/code_context/past_prs/indexes/*.jsonl` is generated by
  `kg-rebuild-graph` as the source-local graph/search index output.
- Live PR-review Pi sessions are stored under
  `decomp-orchestrator/.pi-sessions/pr-review/` for local visibility and are
  ignored by git.
